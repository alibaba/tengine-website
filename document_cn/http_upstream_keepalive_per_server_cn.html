ngx_http_upstream_keepalive_module每server一长连接池

nginx中每个upstream共用一个长连接池，Tengine为keepalive指令新增加了per_server这个参数，它可以为upstream中每个不同的server（ip:port）设置连接池。 

    upstream backend {
        server 127.0.0.1:8080;
        keepalive 32 per_server;
    }

指令
Syntax: keepalive time per_server
Default: -
Context: upstream

指定per_server参数之后，Tengine将为该upstream的每个不同server单独建立连接池。

新增变量
upstream_cached_connection

当于后端建立连接时，如果复用了连接池中的长连接则取值为1，否则值为0。如果与后端建立了多次连接，不同连接的值之间和$upstream_addr变量类似的采用逗号或者冒号分开。
