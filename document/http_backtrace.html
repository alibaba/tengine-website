ngx_http_backtrace_module

The backtrace module is not enabled by default. You should add configure argument to enable it:
--with-backtrace_module.

This module can print the backtrace while a worker process exits when receiving some signals (inclued SIGABR, SIGBUS, SIGFPE, SIGILL, SIGIOT, SIGSEGV). It's useful for debuging when your process exits abnormally.

This module requires backtrace function in libc, of course, you can not enable it in some systems lack of this function, including FreeBSD, Darwin.

Directive
Syntax: backtrace_log log_path
Default: backtrace_log error.log
Context: main

Specify the log file name of backtrace. 

    backtrace_log /path/to/backtrace.log

Syntax: backtrace_max_stack_size size
Default: backtrace_max_stack_size 30
Context: main

Specify the maximum stack depth for backtrace
