Simple example

This is a simple example to compile and install Tengine:
$./configure
$make
$sudo make install

By default, it's installed to /usr/local/nginx. You can use the '--prefix' option to define the specific installation directory.
Configure script options

Most of the options are compatible with Nginx. Here we just lists the specific options in Tengine. If you want to know all the options supported in Tengine, you can run this command './configure --help' for help.

  • --dso-path

    • sets the installation directory for the DSO modules.

  • --dso-tool-path

    • sets the installation pathname of the dso_tool script.

  • --without-dso

    • disable the feature of dynamic shared module in Tengine.

  • --with-jemalloc

    • enable Tengine to link the jemalloc library for memory management.

  • --with-jemalloc=path

    • sets the path to the source of the jemalloc library.
    Make targets

    Most of the targets are compatible with Nginx. Here just lists the specific targets in Tengine.

  • make test

    • run the test cases of Tengine. You need install perl to run this test.

  • make dso_install

    • It will copy the shared module library files to the destination directory which is specified by '--dso-path'. By default, it's in the directory named modules under the Tengine installation directory.
