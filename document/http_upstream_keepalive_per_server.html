ngx_http_upstream_keepalive_module connection pool for every single server address in upstream

Nginx only create one connection pool for each upstream. But Tengine added a new parameter (per_server) for the directive - keepalive, it is able to create a different pool for every single server in the upstream. 

    upstream backend {
        server 127.0.0.1:8080;
        keepalive 32 per_server;
    }

directive
Syntax: keepalive time per_server
Default: -
Context: upstream

While using the parameter - per_server in directive keepalive, Tengine will ctreate a different pool for every single server in this upstream. The size of each pool is 32 in the example.

variable
upstream_cached_connection

keeps the results that whether a connection was got from the connected pool, if so the value is equal to 1, otherwise 0. Several results are separated by commas and colons like in the $upstream_addr variable.
